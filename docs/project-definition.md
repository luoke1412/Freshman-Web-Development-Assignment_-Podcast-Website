# 听见成长项目定义

## 1. 项目名称

听见成长

副标题：一个记录学习、生活与成长灵感的个人播客网站

## 2. 项目定位

这是一个静态多页面个人网站，主题为成长型播客展示。网站内容积极向上，强调学习、阅读、时间管理和校园生活感悟，适合作为网页设计课程小作业。

## 3. 设计目标

1. 满足课程对多页面网站的要求
2. 体现文字、图片、音频、表单等元素的综合运用
3. 使用 `DIV + CSS` 完成页面布局
4. 风格简洁温暖，适合答辩展示
5. 控制实现难度，便于在短时间内完成

## 4. 目标用户

- 大学生
- 喜欢成长类内容的年轻人
- 对播客、阅读和学习方法感兴趣的浏览者

## 5. 页面命名说明

页面名称分为两层：

1. 中文页面名称：用于导航栏、答辩讲解、页面标题
2. 英文文件名：用于实际 HTML 文件命名

对应关系如下：

| 中文名称 | 英文文件名 | 页面作用 |
| --- | --- | --- |
| 首页 | `index.html` | 网站主入口，重点展示页面 |
| 关于我 | `about.html` | 个人简介页面 |
| 节目列表 | `episodes.html` | 展示播客节目卡片 |
| 节目详情 | `episode-detail.html` | 展示单期节目和音频播放器 |
| 成长资源 | `resources.html` | 展示书单、方法、推荐内容 |
| 联系我 | `contact.html` | 表单页面，供用户留言 |

## 6. 功能范围

### 必做内容

1. 至少 5 个页面
2. 首页重点设计
3. 一个个人简介页面
4. 一个表单页面
5. 页面中包含文字、超链接、图像、列表、表单、音频元素

### 本项目最终采用 6 个页面

1. 首页
2. 关于我
3. 节目列表
4. 节目详情
5. 成长资源
6. 联系我

## 7. 内容主题

网站内容围绕以下 4 个方向展开：

1. 学习方法
2. 阅读分享
3. 时间管理
4. 校园成长感悟

这套内容主题具有积极性、实用性和统一性，适合课程作业展示。

## 8. 参考网站与借鉴范围

参考网站：`https://www.mypodops.com/`

借鉴内容：

1. 清晰的顶部导航结构
2. 首页首屏主标题 + 按钮 + 视觉图布局
3. 节目卡片式展示方式
4. 留白充足的现代简洁风格

本项目采用简化设计，只保留适合课程作业的静态页面结构。

## 10. 项目目录规划

```text
podcast-site/
├─ html/
│  ├─ index.html
│  ├─ about.html
│  ├─ episodes.html
│  ├─ episode-detail.html
│  ├─ resources.html
│  └─ contact.html
├─ css/
│  ├─ style.css
│  ├─ index.css
│  ├─ about.css
│  ├─ episodes.css
│  ├─ resources.css
│  └─ contact.css
├─ img/
├─ audio/
├─ docs/
│  ├─ project-definition.md
│  └─ wireframe.md
└─ README.md
```

说明：

1. 所有 HTML 页面统一放在 `html/` 文件夹里，结构会更清楚
2. `css/` 存放公共样式和页面独立样式
3. `img/` 存放头像、封面图、装饰图
4. `audio/` 存放 1 到 3 个 MP3 试听片段
5. `docs/` 存放项目说明文档和原型说明

## 11. 分支规划

- `main`：项目总览和说明书主入口
- `arch`：架构代码、原型说明、基础页面骨架
- `dev`：完整源码和完整说明书

答辩时可说明：项目采用阶段化分支管理，用于区分项目说明、网站架构和完整开发版本。

## 12. 开发流程

### 第一阶段：需求分析

明确网站主题、目标用户、页面数量和功能范围。

### 第二阶段：原型设计

先画首页、关于我页、联系我页的低保真原型图。

### 第三阶段：项目搭架子

创建页面文件、样式文件夹和资源文件夹，先完成公共导航和页脚。

### 第四阶段：完成首页

依次完成：

1. 导航栏
2. 主视觉区
3. 精选节目区
4. 音频试听区
5. 成长资源区
6. 页脚

### 第五阶段：完成其余页面

依次完成：

1. 关于我
2. 节目列表
3. 节目详情
4. 成长资源
5. 联系我

### 第六阶段：统一美化和调试

检查链接、路径、音频播放、页面布局和素材体积。

### 第七阶段：整理项目说明书

形成可答辩的项目说明文档。

## 13. 答辩可用说明

### 为什么选择这个题目

我选择了个人播客网站，因为它能自然结合文字、图片、音频和表单元素，同时内容积极向上，适合展示网页综合设计能力。

### 为什么只用 HTML 和 CSS

本项目以静态页面设计为主，重点考查网页布局、页面美化和多页面组织，因此采用 `HTML + CSS` 即可完成主要目标，也更符合本次小作业的难度控制。

### 网站亮点

1. 页面结构完整
2. 首页重点突出
3. 内容方向积极实用
4. 风格简洁统一
5. 满足课程全部基本要求
